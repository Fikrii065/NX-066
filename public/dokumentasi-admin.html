Voucher Diskon

Manajemen voucher & kode promo untuk pelanggan

Modul voucher memungkinkan admin mengelola kode diskon atau voucher platform yang ditampilkan kepada pengguna. Setiap voucher memiliki ikon, kategori, dan URL tujuan yang dapat dikustomisasi.

  • Tambah Voucher — Buat voucher baru dengan kode unik, nama, ikon (emoji atau gambar base64), kategori, dan URL tujuan klik.
  • Edit Voucher — Perbarui informasi voucher termasuk nama, ikon, link URL, kategori, urutan tampil, dan status aktif/nonaktif.
  • Hapus Voucher — Hapus voucher yang sudah tidak berlaku dari sistem.
  • Aktif/Nonaktif — Toggle status is_active untuk mengontrol visibilitas voucher di halaman publik.
  • Upload Ikon — Mendukung upload gambar ikon dalam format base64 untuk tampilan yang lebih profesional.
ℹ️ Voucher yang aktif (is_active = 1) akan muncul di halaman publik dan diurutkan berdasarkan nilai sort_order. Voucher nonaktif hanya terlihat di panel admin melalui endpoint /api/vouchers/all.
MethodEndpointAksi
GET/api/vouchersDaftar voucher aktif (publik)
GET/api/vouchers/allSemua voucher termasuk nonaktif (admin)
POST/api/vouchersTambah voucher baru
PUT/api/vouchers/:idUpdate voucher
DELETE/api/vouchers/:idHapus voucher

Mutasi Saldo

Riwayat topup, pemakaian, dan penyesuaian saldo pengguna

Admin dapat memantau dan mengelola saldo akun pengguna secara langsung. Setiap perubahan saldo tercatat di tabel balance_logs untuk keperluan audit dan transparansi.

➕ Tambah Saldo
Admin bisa menambah saldo pengguna secara manual dengan catatan alasan untuk rekam jejak.
➖ Kurangi Saldo
Pengurangan saldo disertai validasi bahwa saldo pengguna mencukupi sebelum diproses.
📜 Log Transaksi
Semua perubahan saldo dicatat dengan type (topup/deduct), jumlah, keterangan, dan referensi.
🔒 Transaksi Aman
Setiap operasi saldo menggunakan database transaction (BEGIN/COMMIT/ROLLBACK) untuk mencegah data korup.
⚠️ Operasi saldo menggunakan row-level locking (FOR UPDATE) untuk mencegah race condition saat ada transaksi bersamaan. Selalu sertakan parameter note untuk keperluan audit.
MethodEndpointAksi
GET/api/balance/meCek saldo pengguna (user sendiri)
GET/api/balance/methodsDaftar metode pembayaran untuk topup saldo
POST/api/balance/topupBuat transaksi topup saldo via TokoPay
PATCH/api/admin/users/:id/balanceAdmin tambah/kurangi saldo pengguna

Auto Refund Saldo

Pengembalian saldo otomatis untuk transaksi gagal

Sistem auto refund diaktifkan saat top-up ke Digiflazz gagal (status failed) pada order yang dibayar dengan saldo akun. Saldo dikembalikan secara otomatis tanpa intervensi manual.

  • Trigger Otomatis — Refund dipicu oleh webhook dari Digiflazz ketika status top-up berubah menjadi failed.
  • Validasi Pembayaran — Hanya order dengan payment_method = balance yang mendapat refund ke saldo akun.
  • Update Status Order — Status topup_status diubah menjadi refunded setelah saldo dikembalikan.
  • Log Balance — Pengembalian saldo tercatat di balance_logs dengan referensi order_id.
  • Notifikasi WA — Pengguna mendapat notifikasi WhatsApp otomatis saat refund berhasil diproses.
✅ Alur refund: Webhook Digiflazz → validasi status failed → cek payment_method = balance → kembalikan saldo → update order status → kirim notif WA → catat di balance_logs.

Flash Sale

Produk dengan harga spesial terbatas waktu

Fitur flash sale memungkinkan admin menandai paket tertentu sebagai produk unggulan (is_hot = 1) yang ditampilkan secara khusus di halaman depan dengan badge visual yang menarik perhatian.

  • Tandai Produk Hot — Admin mengaktifkan flag is_hot pada paket tertentu untuk menampilkan badge flash sale.
  • Harga Khusus — Penetapan harga jual (sell_price) berbeda dari harga normal dengan markup lebih rendah.
  • Tampil Prioritas — Produk hot ditampilkan di posisi teratas atau mendapat highlight visual khusus di UI.
  • Sync Digiflazz — Harga base dapat disinkronkan dari Digiflazz untuk memastikan margin tetap terjaga.
ℹ️ Saat ini flag is_hot dikelola melalui endpoint paket. Untuk flash sale berbasis waktu (dengan tanggal mulai/selesai), perlu penambahan kolom flash_start dan flash_end di tabel packages.

Sistem Keanggotaan

Level pelanggan: Basic, Gold, Platinum (Reseller)

Sistem keanggotaan mengatur harga yang diterima pengguna berdasarkan role akun mereka. Saat ini terdapat dua tier utama yang sudah aktif di backend.

👤 Member (Basic)
Harga normal dengan markup persen + minimum. Default untuk semua pengguna baru yang mendaftar.
⭐ Reseller (Gold/Platinum)
Harga tanpa markup (0% + Rp0 minimum). Khusus akun yang diupgrade admin ke role reseller.
  • Atur Level — Admin dapat mengubah role pengguna antara member dan reseller kapan saja.
  • Harga Otomatis — Saat checkout, sistem otomatis mendeteksi JWT token untuk menentukan harga yang berlaku.
  • Notif Upgrade — Pengguna mendapat WhatsApp otomatis saat akun diupgrade ke reseller.
  • Reset Password — Admin bisa reset password via manual atau kirim link token (berlaku 1 jam) ke WhatsApp pengguna.
  • Toggle Akun — Aktifkan/nonaktifkan akun pengguna untuk membatasi akses login.
MethodEndpointAksi
GET/api/admin/usersDaftar pengguna dengan filter & pagination
GET/api/admin/users/:idDetail pengguna + riwayat order
PATCH/api/admin/users/:id/roleUbah role (member/reseller)
PATCH/api/admin/users/:id/toggleAktif/nonaktifkan akun
POST/api/admin/users/:id/reset-passwordReset password (manual atau token WA)

Kelola Admin

Manajemen akun administrator sistem

Admin superadmin dapat mengelola akun admin lainnya. Terdapat tiga level role admin yang berbeda tingkat aksesnya di sistem.

👑 Superadmin
Akses penuh ke semua fitur termasuk kelola admin lain dan pengaturan sistem kritis.
🛡️ Admin
Akses standar untuk mengelola produk, order, pengguna, dan konten.
🎧 CS (Customer Service)
Akses terbatas untuk membantu penanganan order dan pertanyaan pelanggan.
🔑 Autentikasi JWT
Login admin menggunakan JWT token terpisah dari JWT pengguna biasa (type: admin vs user).
  • Tambah Admin — Buat akun admin baru dengan username, email, password, dan role yang ditentukan.
  • Edit Admin — Perbarui informasi akun admin termasuk perubahan role.
  • Hapus Admin — Hapus akun admin yang tidak aktif lagi dari sistem.
  • Password Hashing — Semua password admin di-hash menggunakan bcrypt dengan salt rounds 12.

Kelola Kategori (Game)

Manajemen game/kategori produk top-up

Kategori dalam konteks NX-066 adalah daftar game yang tersedia untuk di-top-up. Admin dapat mengelola game yang ditampilkan beserta konfigurasinya.

  • Tambah Game — Daftarkan game baru dengan kode unik, nama, ikon, parameter input (userId/zoneId), dan kode VIP Digiflazz.
  • Edit Game — Perbarui detail game termasuk ikon kustom (base64 image), urutan tampil, dan status aktif.
  • Hapus Game — Hapus game beserta semua paket yang terkait (CASCADE DELETE).
  • Aktif/Nonaktif — Toggle is_active untuk menyembunyikan game dari halaman publik tanpa menghapusnya.
  • Sync Produk Digiflazz — Sinkronisasi otomatis produk dari price list Digiflazz berdasarkan kecocokan nama brand/game.
MethodEndpointAksi
GET/api/admin/digiflazz/pricelistAmbil price list dari Digiflazz API
POST/api/admin/digiflazz/syncSinkronisasi produk pilihan ke database
POST/api/admin/sync-productsQuick sync semua produk Digiflazz ke game

Kelola Akun Pengguna

Manajemen akun member dan reseller

Modul ini memberikan admin visibilitas dan kontrol penuh atas akun pengguna yang terdaftar, termasuk riwayat transaksi dan pengelolaan saldo.

  • Tambah Akun — Pengguna bisa mendaftar sendiri, atau admin dapat membuat akun langsung.
  • Edit Akun — Perbarui nama, email, nomor telepon, atau role pengguna.
  • Hapus Akun — Hapus permanen akun pengguna dari database.
  • Cari & Filter — Pencarian berdasarkan nama, email, atau nomor HP dengan filter role (member/reseller).
  • Lihat Riwayat Order — Tampilkan 20 order terakhir beserta total pembelian per pengguna.
  • Manajemen Saldo — Tambah atau kurangi saldo pengguna dengan catatan alasan.

Kelola Pembelian

Manajemen transaksi order pelanggan

Admin dapat memantau dan mengintervensi transaksi yang bermasalah. Setiap order memiliki dua status yang dikelola secara independen: status pembayaran dan status top-up.

💰 Status Pembayaran
unpaid → paid → expired/failed. Diperbarui via webhook TokoPay secara otomatis.
🎯 Status Top-up
pending → processing → success/failed/refunded. Diperbarui via webhook Digiflazz.
  • Edit Order — Admin bisa memperbarui status order secara manual untuk kasus penanganan khusus.
  • Hapus Order — Hapus data order yang tidak valid atau duplikat dari sistem.
  • Retry Top-up — Coba ulang pengiriman ke Digiflazz untuk order yang gagal diproses.
  • Cek Nickname — Verifikasi username game pengguna sebelum memproses top-up manual.
  • Lihat Log — Akses topup_logs dan payment_logs untuk debug transaksi bermasalah.
MethodEndpointAksi
GET/api/admin/check-nicknameCek nickname game via Digiflazz
POST/api/orders/createBuat order baru
GET/api/orders/check/:idCek status order pelanggan

Kelola Metode Pembayaran

Manajemen channel pembayaran via TokoPay

Sistem mendukung 10 metode pembayaran yang dapat diaktifkan/dinonaktifkan secara independen. Setiap metode memiliki logo, biaya layanan, dan channel khusus yang dikonfigurasi dari settings.

⚡ QRIS Realtime
Pembayaran via QRIS dengan konfirmasi real-time. Biaya dapat dikonfigurasi per transaksi.
📱 E-Wallet
DANA, OVO, GoPay, ShopeePay dengan biaya layanan yang dapat diatur masing-masing.
🏦 Virtual Account
BCA, BNI, BRI, Mandiri, Permata dengan nomor VA yang unik per transaksi.
💰 Saldo Akun
Pembayaran dari saldo akun internal. Tanpa biaya tambahan, proses instan.
  • Aktif/Nonaktif Metode — Toggle pay_enabled_[KODE] untuk masing-masing metode pembayaran.
  • Atur Biaya Layanan — Tetapkan fee_[KODE] (Rupiah) untuk biaya per metode pembayaran.
  • Upload Logo — Simpan logo metode pembayaran dalam format base64 yang ditampilkan di UI checkout.
  • Konfigurasi Channel — Mapping kode internal ke kode channel TokoPay via pay_channel_[KODE].
MethodEndpointAksi
GET/api/settingsAmbil setting publik termasuk fee & status metode
GET/api/settings/allSemua setting termasuk kredensial (admin)
PUT/api/settingsUpdate setting (whitelist key)

Notifikasi WhatsApp (Fonnte)

Kirim pesan otomatis ke pelanggan via WhatsApp

Sistem notifikasi menggunakan layanan Fonnte untuk mengirim pesan WhatsApp otomatis ke pengguna pada berbagai event penting dalam perjalanan transaksi.

✅ Order Berhasil
Konfirmasi top-up sukses dengan detail order, game, paket, dan serial number jika ada.
❌ Order Gagal
Pemberitahuan top-up gagal beserta informasi refund saldo jika pembayaran via saldo.
↩️ Refund Saldo
Notifikasi pengembalian saldo dengan jumlah yang dikembalikan dan nomor order referensi.
🔑 Reset Password
Kirim link reset password atau password baru yang telah di-reset admin kepada pengguna.
⭐ Upgrade Reseller
Selamat datang sebagai reseller lengkap dengan informasi keuntungan harga yang didapat.
💳 Topup Saldo
Konfirmasi topup saldo berhasil dengan jumlah yang masuk dan saldo akhir akun.
⚠️ Notifikasi WA memerlukan konfigurasi fonte_token dan fonte_device di tabel settings. Aktifkan fitur dengan mengatur feature_whatsapp_notif = 1. Semua notifikasi bersifat opsional — kegagalan pengiriman WA tidak menghentikan proses utama.

Kelola Sosial Media

Link sosmed & informasi kontak toko

Admin dapat mengatur informasi kontak dan link media sosial yang ditampilkan di footer atau halaman toko. Semua disimpan sebagai key-value di tabel settings.

  • WhatsApp Kontak — Nomor WhatsApp CS yang muncul sebagai tombol chat langsung.
  • Email Kontak — Alamat email customer service untuk komunikasi formal.
  • Instagram — Link profil Instagram toko untuk keperluan promosi.
  • Telegram — Link grup atau channel Telegram untuk komunitas pelanggan.
  • Footer Text — Teks hak cipta/copyright yang muncul di bagian bawah halaman.
ℹ️ Semua setting sosmed dikelola via endpoint PUT /api/settings dengan key: contact_wa, contact_email, social_instagram, social_telegram, footer_text. Update dilakukan dalam satu request batch.

Kelola Syarat & Ketentuan

Konten legal dan informasi toko

Admin dapat mengelola berbagai konten informatif dan pengaturan tampilan toko yang mempengaruhi pengalaman pengguna secara keseluruhan.

  • Nama & Tagline Toko — Identitas toko (site_name, site_tagline) yang muncul di judul halaman.
  • Logo Toko — Upload logo toko dalam format gambar base64 atau URL eksternal.
  • Warna Tema — Ubah warna aksen utama (accent_color) untuk personalisasi tampilan UI.
  • Mode Maintenance — Aktifkan feature_maintenance = 1 untuk menutup toko sementara saat ada update.
  • Guest Checkout — Toggle feature_guest_checkout untuk mengizinkan/melarang pembelian tanpa login.
  • Check Nickname — Aktifkan/nonaktifkan fitur verifikasi username game sebelum checkout.
✅ Setting disimpan menggunakan mekanisme INSERT ... ON DUPLICATE KEY UPDATE sehingga aman dijalankan berulang kali. Hanya key yang ada dalam whitelist yang diizinkan untuk mencegah penyimpanan data tidak sah.
